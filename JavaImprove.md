## 🎯 一、学习目标定位

你不是零基础入门者，你的目标应是：

> **把“计算机系统知识”内化成能指导架构设计与性能优化的底层逻辑。**

换句话说：

* 不只是“知道 TCP 三次握手”，而是能解释“为什么高并发下要调内核参数、为什么 Java Socket 会阻塞”。
* 不只是“知道进程线程区别”，而是能理解“Java 线程池和操作系统调度器的关系”。
* 不只是“知道内存结构”，而是能推导“JVM、CPU Cache、虚拟内存三者如何影响性能”。

## 📘 三、详细学习路径

### **阶段 1：计算机系统全景认知**

**目标：** 搭建知识地图，了解各个子领域如何协同。

📚 推荐学习路径：

* 《计算机是怎样跑起来的》（入门级）
* 《深入理解计算机系统（CSAPP）》前 3 章：信息表示、程序编译与执行
* 《程序员的自我修养：链接、装载与库》

📌 学习要点：

* 程序从 `.java` → `.class` → 机器码 → 内存 的全过程；
* 理解编译、链接、加载、执行的关系；
* 初步理解计算机层次结构（CPU、内存、磁盘、操作系统、网络）。

🎯 输出目标：

* 画出一张“从源代码到运行时”的系统图；
* 能解释“一行 Java 代码最终是如何在 CPU 上执行的”。

---

### **阶段 2：操作系统 + 计算机组成原理**

**目标：** 理解程序如何被操作系统调度与运行。

📚 推荐书籍：

1. 《操作系统：精髓与设计原理》或《现代操作系统》（Tanenbaum）
2. 《计算机组成与设计（MIPS 版）》或《深入理解计算机系统》第 4～9 章

📌 核心主题：

* 进程、线程、内核态与用户态；
* 调度算法、同步、死锁；
* 内存管理（虚拟内存、分页、段页式）；
* 文件系统；
* CPU 架构、流水线、Cache、指令执行。

💡 实践建议：

* 在 Linux 上手动查看进程、线程、文件描述符：

  ```bash
  ps -ef
  top
  strace java HelloWorld
  lsof -p <pid>
  ```
* 用 Java 写一个简单的“线程调度模拟器”（体现进程阻塞、调度、时间片轮转）。

🎯 输出目标：

* 理解操作系统与 JVM 之间的关系；
* 能解释“为什么 Java 线程不一定一一对应操作系统线程”。

---

### **阶段 3：计算机网络 + 分布式系统**

**目标：** 从网络协议到分布式通信栈全面掌握。

📚 推荐书籍：

1. 《计算机网络（谢希仁）》或《Computer Networking: A Top-Down Approach》
2. 《图解HTTP》《TCP/IP详解 卷1》
3. 《分布式系统原理与范型》《Designing Data-Intensive Applications (DDIA)》

📌 核心主题：

* OSI 七层模型；
* TCP/UDP、HTTP、TLS；
* 网络拥塞控制、流量控制；
* RPC、消息队列、负载均衡、CAP、Raft 等。

💡 实践建议：

* 用 Java 实现一个简易的 HTTP 服务器（用 Socket）；
* 抓包分析三次握手；
* 分析 Netty、gRPC、Dubbo 的底层网络通信机制。

🎯 输出目标：

* 能解释“为什么 TCP 会粘包”“NIO 是怎么工作的”“为什么 HTTP2 更高效”；
* 掌握网络层到分布式系统层的全栈通信链路。

---

### **阶段 4：编译原理 + 算法 + 性能优化**

**目标：** 具备系统级分析与优化能力。

📚 推荐书籍：

1. 《编译原理（龙书）》或《Crafting Interpreters》（更实战）
2. 《算法导论》《算法图解》《数据结构与算法之美》
3. 《高性能 MySQL》《Java 性能权威指南》《系统性能分析实战》

📌 学习重点：

* 编译器如何将语言转为字节码；
* Java JIT / AOT；
* 常见算法与数据结构优化；
* 性能分析方法论（CPU / 内存 / I/O / GC）。

💡 实践建议：

* 写一个简单的表达式解释器；
* 使用 `perf`、`async-profiler` 分析 Java 应用；
* 优化自己项目中最慢的一段代码。

🎯 输出目标：

* 掌握性能分析的科学方法；
* 具备“代码–系统–硬件”的贯通视角。
